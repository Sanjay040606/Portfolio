SANJAY PORTFOLIO PROJECT EXPLANATION

1. PROJECT PURPOSE

This project is a personal portfolio website for Sanjay S.
Its purpose is to present profile details, projects, skills, history, and contact information
in a clean and premium portfolio format inspired by the reference site.


2. TOOLS USED

1. Next.js
Why we need it:
- Next.js gives a proper project structure for modern web applications.
- It supports production builds, routing, metadata handling, and optimized static rendering.
- It is better than a plain HTML file for future scaling and deployment.

2. React
Why we need it:
- React helps build the page using reusable UI sections and structured component logic.
- It makes content easier to manage and update later.

3. Tailwind CSS
Why we need it:
- Tailwind allows fast styling directly in the component files.
- It helps create a custom layout without writing large separate CSS systems.
- It is very useful for responsive design and spacing control.

4. GSAP
Why we need it:
- GSAP is used for smooth animations on load.
- It helps create the premium motion feel requested for the portfolio.
- It is more flexible and smoother for animation control than plain CSS in many cases.

5. TypeScript
Why we need it:
- TypeScript improves code safety.
- It helps catch mistakes early during development.
- Next.js works very well with TypeScript by default.

6. Next/Image
Why we need it:
- It is used to render the profile image in an optimized Next.js-friendly way.
- It helps keep image handling cleaner inside the app.

7. Mailto contact handling
Why we need it:
- Right now it allows contact form submission without building a backend.
- It is a simple starting solution until a real mail service is added.


3. TOOLS NOT YET USED BUT RECOMMENDED LATER

1. Resend / EmailJS / Formspree
Why we may need them:
- These tools allow a real working contact form.
- Messages can be sent directly from the website without opening the user's mail app.

2. Vercel
Why we may need it:
- Vercel is the easiest platform to deploy a Next.js app.
- It gives a live hosted portfolio link quickly.

3. Three.js
Why we may need it:
- Three.js can be added if you want advanced WebGL effects similar to high-end portfolio sites.
- It is optional because it increases project complexity.

4. Figma
Why we may need it:
- Figma is useful for planning layout, spacing, and visual hierarchy before coding.
- It helps if you want a more exact visual system before another design pass.


4. HOW THE PROJECT WORKS

1. app/layout.tsx
- This is the root layout file.
- It wraps the whole application.
- It also sets the page metadata like title and description.

2. app/page.tsx
- This is the main portfolio page.
- It contains:
  HOME
  MY STORY
  PROJECTS
  HIGHLIGHTS
  HISTORY
  CONTACT
- It also contains the GSAP animation logic and the mailto form behavior.

3. app/globals.css
- This file loads Tailwind CSS.
- It defines the global background, text color, font styles, and shared base styles.

4. public folder
- This folder stores static assets.
- The profile image is stored here.
- The resume PDF is stored here.
- Anything in this folder can be accessed directly in the app.

5. Contact form working
- When the user fills the form and clicks submit, JavaScript collects the values.
- The form builds a mailto link using the entered name, email, subject, and message.
- Then it opens the user's default mail application.
- This means the form is visually complete but not yet connected to a backend mail service.

6. Animation working
- GSAP runs when the page loads.
- It animates text blocks, lines, and the image.
- This creates a smoother premium entrance effect.
